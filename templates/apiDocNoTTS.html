📈 Check Quota

GET /api/quota

Check your remaining quota available for generation tasks.

10
Maximum Concurrent Tasks Up to 10 tasks can run simultaneously. Check available_slots in the quota response to know how many slots are free before submitting new tasks. 
curl -X GET http://127.0.0.1:5000/api/quota \
  -H "Authorization: Bearer YOUR_API_KEY"
import requests

url = "http://127.0.0.1:5000/api/quota"
headers = {"Authorization": "Bearer YOUR_API_KEY"}

response = requests.get(url, headers=headers)
print(response.json())

Success Response (200 OK)

{
  "quota": 42,
  "running_tasks": 1,
  "max_concurrent": 10,
  "available_slots": 9
}
⚠️ Important Quota Information:

Every successful API operation consumes 1 Quota unit. This applies to:

  • Image Generation (Text-to-Image or Image-to-Image)
  • Video Generation (Text-to-Video or Image-to-Video)
  • Text-to-Speech Generation

Once the quota reaches 0, new generation requests will fail.

🖼 Image Generation

POST /api/generate/image

Generate images from text prompts or transform existing images.

Request Body

Parameter Type Required Description
prompt string Required Text description of the desired image. Max 4000 characters.
reference_images array Optional Array of base64 encoded images for img2img transformation. Multiple character references supported. Max 5 images.
model string Optional Model version (default: NANO BANANA PRO)
Options: NANO_BANANA_PRO, NANO_BANANA, NANO_BANANA_2, GPT_IMAGE_2
resolution string Optional Only for NANO BANANA PRO, NANO BANANA 2, and GPT IMAGE 2 models: 1K, 2K, 4K (default: 2K)
Not applicable for NANO BANANA model
size string Optional For all image models:
16:9, 9:16, 1:1, 3:4, 4:3, 3:2
(default: 16:9)
📌 Model Compatibility:
  • NANO BANANA PRO: Supports resolution (1K, 2K, 4K) and all size options
  • NANO BANANA 2: Supports resolution (1K, 2K, 4K) and all size options
  • NANO BANANA: Only supports size options (no resolution parameter)
  • GPT_IMAGE_2: Supports resolution (1K, 2K, 4K) and all size options

Example Request (NANO BANANA PRO Model — Text-to-Image)

curl -X POST http://127.0.0.1:5000/api/generate/image \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "a futuristic city with neon lights, ultra realistic, 4k",
    "model": "NANO_BANANA_PRO",
    "resolution": "2K",
    "size": "16:9"
  }'

🎥 Video Generation

POST /api/generate/video

Generate videos from text prompts or animate existing images using SORA_2, VEO_3, VIDU_Q3, or SEEDANCE_2_0 models.

Request Body

Parameter Type Required Description
prompt string Required Text description of the desired video/animation. Max 2000 characters.
model string Optional Video generation model:
• SORA_2 (default) — 10 seconds, 720p
• VEO_3 — 8 seconds, 720p
• VIDU_Q3 — image-to-video only; 720p/5s · 512p/5s · 512p/10s
• QUALITY_V2_5 — image-to-video only; 720p/5s or 480p/10s
• SEEDANCE_2_0 — 5 seconds, 480p; text-to-video, start frame + end frame, or reference images
size string Optional 16:9, 9:16 (default: 16:9). VIDU_Q3: send "AUTO" — size is determined automatically from the input image. SEEDANCE_2_0: supports 16:9, 9:16, 1:1, 4:3, 3:4 (not applicable for image-to-video mode).
duration integer Optional VIDU_Q3 and QUALITY_V2_5 only. Duration in seconds: 5 or 10. Default: 5.
VIDU_Q3: 720p supports only 5s · 512p supports 5s or 10s
QUALITY_V2_5: 720p supports only 5s · 480p supports 5s or 10s
resolution string Optional VIDU_Q3 only. Options: 720p (default), 512p.
Note: 720p only supports 5s duration. 512p supports 5s or 10s.
start_frame string Optional Base64 encoded start frame image for img2video animation
end_frame string Optional VEO_3 and SEEDANCE_2_0 only. Base64 encoded end frame image. Requires start_frame to also be provided.
reference_images array Optional VEO_3 and SEEDANCE_2_0 only. Array of base64 encoded reference images to guide style/content. Max 3 images. Cannot be used together with start_frame or end_frame.
📌 Video Model Info:
  • SORA_2 (default): Duration: 10 seconds, Resolution: 720p. Supports text-to-video and image-to-video (start frame only). Does not support end_frame or reference_images.
  • VEO_3: Duration: 8 seconds, Resolution: 720p. Supports text-to-video, image-to-video (start frame), start+end frame, and reference images.
  • VIDU_Q3: Image-to-video only. Resolution: 720p (5s only) or 512p (5s or 10s). Requires size="AUTO". Does not support end_frame or reference_images.
  • QUALITY_V2_5: Image-to-video only. Duration: 5s (720p) or 10s (480p). Requires size="AUTO". Does not support end_frame or reference_images.
  • SEEDANCE_2_0: Duration: 5 seconds, Resolution: 480p. Supports text-to-video, start frame + end frame, and reference images (max 3). Supported sizes: 16:9, 9:16, 1:1, 4:3, 3:4.
  • Supported Sizes: 16:9, 9:16 (SORA_2 & VEO_3) · AUTO (VIDU_Q3 & QUALITY_V2_5) · 16:9, 9:16, 1:1, 4:3, 3:4 (SEEDANCE_2_0)

🎙 Text-to-Speech

POST /api/generate/tts

Convert text to natural-sounding speech using AI voices.

Request Body

Parameter Type Required Description
text string Required Text to convert to speech
voiceId string Optional Voice ID from /api/tts/voices (default: English_expressive_narrator)
speed float Optional Speech speed: 0.5 – 2.0 (default: 1.0)
pitch integer Optional Pitch: -12 – 12 (default: 0)
volume float Optional Volume: 0.01 – 10.0 (default: 1.0)
model string Optional MINIMAX-TURBO (default) or MINIMAX
emotion string Optional Emotion: auto (default), happy, sad, angry, fearful, disgusted, surprised, calm, fluent

Example Request (All Parameters)

curl -X POST http://127.0.0.1:5000/api/generate/tts \

🎵 Music Generation

POST /api/generate/music

Generate high-quality music and audio from text prompts or reference audio.

Request Body

Parameter Type Required Description
prompt string Required Description of the music to generate.
model string Optional Model version (default: SUNO)
Options: SUNO
style string Optional Music style (e.g., "lo-fi", "cinematic", "rock").
lyrics string Optional Lyrics for the song.
main_mode string Optional Generation mode: Vocal (default) or Instrumental. When Instrumental, lyrics is ignored and instrumental music is generated.
mode string Optional Generation mode (default: TEXT):
• TEXT — generate from prompt only
• COVER — create a cover of reference audio (requires audio_base64)
• ADD_INSTRUMENTAL — add instrumental to reference audio (requires audio_base64)
• ADD_VOCAL — add vocals to reference audio (requires audio_base64)
audio string Optional Base64 encoded reference MP3 audio. Required when mode is COVER, ADD_INSTRUMENTAL, or ADD_VOCAL.

Example Request

curl -X POST http://127.0.0.1:5000/api/generate/music \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "a chill lo-fi beat for studying",
    "style": "lo-fi",
    "lyrics": "",
    "main_mode": "Instrumental",
    "mode": "TEXT"
  }'
import requests

Pagination (Optional)

By default /api/status returns all tasks. Add page and per_page query parameters to paginate results.

Parameter Type Default Description
page integer — Page number (starting from 1). If omitted, all tasks are returned without pagination.
per_page integer 6 Number of tasks per page. Only used when page is provided.

Paginated Response (200 OK)

// GET /api/status?page=1&per_page=6
{
  "tasks": [ /* ... */ ],
  "running_tasks": 1,
  "max_concurrent": 10,
  "page": 1,
  "per_page": 6,
  "total": 42,
  "total_pages": 7
}

Best Practices

  • Poll task status every 10 seconds for image tasks
  • Poll task status every 5–10 seconds for video tasks
  • Image generation takes approximately 30–120 seconds
  • Video generation takes approximately 60–300 seconds
  • Always check quota before submitting multiple tasks
  • Store task IDs for later retrieval
